## RNN

### N-gram

不必回溯一个词的全部历史，可以近似地认为，一个词的出现概率只与它前面有限的 n−1 个词有关，如图3.1所示。基于这个假设建立的语言模型，我们称之为 **N-gram模型**。在大型语料库中进行**最大似然估计(Maximum Likelihood Estimation,MLE)** 来计算

N-gram 模型虽然简单有效，但有两个致命缺陷：

1. **数据稀疏性 (Sparsity)** ：如果一个词序列从未在语料库中出现，其概率估计就为 0，这显然是不合理的。虽然可以通过平滑 (Smoothing) 技术缓解，但无法根除。
2. **泛化能力差：**模型无法理解词与词之间的语义相似性。例如，即使模型在语料库中见过很多次 `agent learns`，它也无法将这个知识泛化到语义相似的词上。当我们计算 `robot learns` 的概率时，如果 `robot` 这个词从未出现过，或者 `robot learns` 这个组合从未出现过，模型计算出的概率也会是零。模型无法理解 `agent` 和 `robot` 在语义上的相似性。

### RNN

**循环神经网络 (Recurrent Neural Network, RNN)** 应运而生，其核心思想非常直观：<u>为网络增加“记忆”能力</u>。RNN 的设计引入了一个**隐藏状态 (hidden state)** 向量，我们可以将其理解为网络的短期记忆。在处理序列的每一步，网络都会读取当前的输入词，并结合它上一刻的记忆（即上一个时间步的隐藏状态），然后生成一个新的记忆（即当前时间步的隐藏状态）传递给下一刻。这个循环往复的过程，使得信息可以在序列中不断向后传递。

标准的 RNN 在实践中存在一个严重的问题：**<u>长期依赖问题 (Long-term Dependency Problem)</u>** 。在训练过程中，模型需要通过反向传播算法根据输出端的误差来调整网络深处的权重。对于 RNN 而言，序列的长度就是网络的深度。当序列很长时，梯度在从后向前传播的过程中会经过多次连乘，这会导致梯度值快速趋向于零（**梯度消失**）或变得极大（**梯度爆炸**）。<u>梯度消失</u>使得模型无法有效学习到序列早期信息对后期输出的影响，即难以捕捉长距离的依赖关系。

RNN的梯度是一个总的梯度和，它的梯度消失并不是变为0，而是**<u>总梯度被近距离梯度主导</u>，对远距离梯度几乎忽略不计（远距离的梯度趋近于0，模型无法学习长期依赖）**

RNN串行计算”的模式有两个致命问题：

1. **并行性差**：无法同时处理多个词，训练速度极慢，面对长句子（如段落、文章）时效率更低；
2. **长距离依赖难捕捉**：当句子过长时，早期词的信息会逐渐“遗忘”，比如翻译“小明昨天买的那本书，今天被他弄丢了”时，RNN很难将“他”和“小明”精准关联。

### LSTM

为了解决长期依赖问题，**长短时记忆网络 (Long Short-Term Memory, LSTM)** 被设计出来。

LSTM 是一种特殊的 RNN，其核心创新在于引入了<u>**细胞状态 (Cell State)** 和一套精密的**门控机制 (Gating Mechanism)**</u> 。细胞状态可以看作是一条独立于隐藏状态的信息通路，允许信息在时间步之间更顺畅地传递。门控机制则是由几个小型神经网络构成，它们可以学习如何有选择地让信息通过，从而控制细胞状态中信息的增加与移除。这些门包括：

- **遗忘门 (Forget Gate)**：决定从上一时刻的细胞状态中丢弃哪些信息。
- **输入门 (Input Gate)**：决定将当前输入中的哪些新信息存入细胞状态。
- **输出门 (Output Gate)**：决定根据当前的细胞状态，输出哪些信息到隐藏状态。

- **遮蔽多头自注意力 (Masked Multi-Head Self-Attention):**
  - **遮蔽 (Masked/Causal Mask):** 这是 GPT-1 作为**自回归模型**的关键。在计算注意力时，一个**因果掩码**会被应用，确保模型在预测当前词时，**只能看到它前面的词，而不能“偷看”后面的词**。
  
  - **多头 (Multi-Head):** 意味着模型并行地执行**多个独立的自注意力计算**。每个“头”学习不同的注意力模式，捕捉不同的关系。例如，一个头可能关注语法依赖，另一个可能关注语义关联。这种并行性提高了模型的表达能力。
  
  - **自注意力 (Self-Attention):** 允许模型在处理序列中的一个词时，同时关注序列中的**所有其他词**。它计算每个词对序列中所有其他词的“重要性”或“相关性”分数。
  
- **层归一化 (Layer Normalization):** 在每个子层（自注意力层和前馈网络层）的输入之前和之后都会应用层归一化。这有助于<u>稳定训练过程，并加速收敛</u>。

- **前馈神经网络 (Feed-Forward Network):** 每个注意力子层的输出会进一步通过一个位置共享（即对序列中每个位置应用相同的网络）的**两层全连接神经网络**。这增加了<u>模型的非线性表达能力，允许它学习更复杂的特征</u>。

- **残差连接 (Residual Connections):** 每个子层（自注意力层和前馈网络层）都使用了残差连接，即<u>子层的输入会直接加到其输出上</u>。这有助于<u>缓解深度网络的梯度消失问题</u>，使模型能够更深地训练。

**Transformer相较传统RNN的优势**

- 并行计算
  - RNN循环依赖上一步输出，无法并行处理序列。
  - 而Transformer的自注意力机制对序列中每个位置同时计算与其他位置的相关性，没有时序依赖，可以用矩阵运算并行处理整句话 。
  - 这使得训练时GPU利用率高，能够在海量数据上训练非常深的模型。
- 长程依赖捕获
  - 注意力机制可以让每个词直接“看”其他所有词，不管距离多远都在一次注意力计算中体现。
  - 而RNN要经过许多步状态传递才能把远处信息带过来，中间可能梯度消失或信息淡化。
- 更好的表示能力
  - 多头注意力提供了对不同关系特征的建模能力，且注意力是输入内容自适应的，加上FFN的非线性变换，整个模型具有极强表达力。
  - RNN虽然也可以堆叠层和门控，但Transformer因为摆脱了序列顺序计算限制，可以更自由地增加层数和宽度（维度），加上FeedForward层提供丰富特征组合，因而更易扩展出超高参数模型。
- 训练稳定性
  - Transformer用了注意力+残差+正则化，使其训练较稳定。
  - 某些RNN需要小心初始化和梯度剪裁，以避免梯度消失和爆炸问题。
    - **梯度爆炸（gradient explosion）**：训练刚开始，Loss 一下子飙升为无穷大；
    - **梯度消失（gradient vanishing）**：Loss 长期不下降，模型几乎不学习。

```
# 导入相关需要的包
import math
import torch
import torch.nn as nn

import warnings
warnings.filterwarnings(action="ignore")

# 写一个 Block
class SimpleDecoder(nn.Module):
    def __init__(self, hidden_dim, nums_head, dropout=0.1):
        super().__init__()

        self.nums_head = nums_head
        self.head_dim = hidden_dim // nums_head

        self.dropout = dropout

        # 这里按照 transformers 中的 decoder 来写，用 post_norm 的方式实现注意有 残差链接
        # eps 是为了防止溢出；其中 llama 系列的模型一般用的是 RMSnorm 以及 pre-norm（为了稳定性）
        # RMSnorm 没有一个 recenter 的操作，而 layernorm 是让模型重新变成 均值为 0，方差为 1
        # RMS 使用 w平方根均值进行归一化 $\sqrt{\frac{1}{n} \sum_{1}^{n}{a_i^2} }$
        self.layernorm_att = nn.LayerNorm(hidden_dim, eps=0.00001)

        self.q_proj = nn.Linear(hidden_dim, hidden_dim)
        self.k_proj = nn.Linear(hidden_dim, hidden_dim)
        self.v_proj = nn.Linear(hidden_dim, hidden_dim)
        self.o_proj = nn.Linear(hidden_dim, hidden_dim)
        self.drop_att = nn.Dropout(self.dropout)

        # for ffn 准备
        self.up_proj = nn.Linear(hidden_dim, hidden_dim * 4)
        self.down_proj = nn.Linear(hidden_dim * 4, hidden_dim)
        self.layernorm_ffn = nn.LayerNorm(hidden_dim, eps=0.00001)
        self.act_fn = nn.ReLU()
        
        self.drop_ffn = nn.Dropout(self.dropout)

    def attention_output(self, query, key, value, attention_mask=None):
        # 计算两者相关性
        key = key.transpose(2, 3)  # (batch, num_head, head_dim, seq)
        att_weight = torch.matmul(query, key) / math.sqrt(self.head_dim)

        # attention mask 进行依次调整；变成 causal_attention
        if attention_mask is not None:
            # 变成下三角矩阵
            attention_mask = attention_mask.tril()
            att_weight = att_weight.masked_fill(attention_mask == 0, float("-1e20"))
        else:
            # 人工构造一个下三角的 attention mask
            attention_mask = torch.ones_like(att_weight).tril()
            att_weight = att_weight.masked_fill(attention_mask == 0, float("-1e20"))

        att_weight = torch.softmax(att_weight, dim=-1)
        print(att_weight)

        att_weight = self.drop_att(att_weight)

        mid_output = torch.matmul(att_weight, value)
        # mid_output shape is: (batch, nums_head, seq, head_dim)

        mid_output = mid_output.transpose(1, 2).contiguous()
        batch, seq, _, _ = mid_output.size()
        mid_output = mid_output.view(batch, seq, -1)
        output = self.o_proj(mid_output)
        return output

    def attention_block(self, X, attention_mask=None):
        batch, seq, _ = X.size()
        query = self.q_proj(X).view(batch, seq, self.nums_head, -1).transpose(1, 2)
        key = self.k_proj(X).view(batch, seq, self.nums_head, -1).transpose(1, 2)
        value = self.v_proj(X).view(batch, seq, self.nums_head, -1).transpose(1, 2)

        output = self.attention_output(
            query,
            key,
            value,
            attention_mask=attention_mask,
        )
        return self.layernorm_att(X + output)

    def ffn_block(self, X):
        up = self.act_fn(
            self.up_proj(X),
        )
        down = self.down_proj(up)

        # 执行 dropout
        down = self.drop_ffn(down)

        # 进行 norm 操作
        return self.layernorm_ffn(X + down)

    def forward(self, X, attention_mask=None):
        # X 一般假设是已经经过 embedding 的输入， (batch, seq, hidden_dim)
        # attention_mask 一般指的是 tokenizer 后返回的 mask 结果，表示哪些样本需要忽略
        # shape 一般是： (batch, nums_head, seq)

        att_output = self.attention_block(X, attention_mask=attention_mask)
        ffn_output = self.ffn_block(att_output)
        return ffn_output


# 测试

### Types of Large Language Models

#### Autoencoder-Based Model

One type of large language model is the autoencoder-based model, which works by encoding input text into a lower-dimensional representation and then generating new text based on that representation. This type of model is especially good for tasks like summarizing text or generating content.

#### Sequence-to-Sequence Model

Another type of large language model is the sequence-to-sequence model, which takes an input sequence (like a sentence) and generates an output sequence (like a translation into another language). These models are often used for machine translation and text summarization.

#### Transformer-Based Models

Transformer-based models are another popular type of large language model. These models use a neural network architecture that's great at understanding long-range dependencies in text data, making them useful for a wide range of language tasks, including generating text, translating languages, and answering questions.

#### Recursive Neural Network Models

Recursive neural network models are designed to handle structured data like parse trees, which represent the syntactic structure of a sentence. These models are useful for tasks like sentiment analysis and natural language inference.

#### Hierarchical Models

Finally, hierarchical models are designed to handle text at different levels of granularity, such as sentences, paragraphs, and documents. These models are used for tasks like document classification and topic modeling.

### Types of LLM Architectures

Not all Transformers are built for the same purpose. Here are simple breakdown:

#### 1. Encoder Only Models (BERT, RoBERTa)

用MLM（Mask Language Model）预训练，不擅长做生成任务，做NLU一般也需要有监督的下游数据微调

Trained to *understand* 理解 text by predicting masked words.
Used for:

- Sentiment analysis
- Entity recognition
- Search ranking
- 如**文本分类BERT**和图片分类ViT（Vision Transformer）

Think of BERT as the *reader* of the AI world.

#### 2. Decoder-Only Models **（也叫Causal-LM，因果语言模型）**(GPT, LLaMA, Mistral)

用Next Token Prediction预训练，全程都是单向语言信息传输

兼顾理解和生成，在各种下游任务上的zero-shot和few-shot（也就是上下文学习，In-context Learning）泛化性能都很好

Trained to *generate* 生成 the next word given the previous ones.
Used for:

- Chatbots (ChatGPT, Claude)
- Story and code generation
- Creative writing

GPT models are the *writers* of the AI world.

#### 3. Encoder–Decoder Models (T5, BART, FLAN-T5)

seq2seq架构，Encoder和Decoder之间天然有一个结构上的分离。Encoder的attention矩阵信息是双向的（互相可看见），而Decoder是单向自回归生成（只能看到自己及之前的），两者之间不共享参数

Trained for *transformation 翻译 tasks* meaning one sequence into another.
Used for:

- Summarization
- Translation
- Question answering

These are the *translators* of AI

#### **Prefix-LM架构（前缀语言模型）：**如UniLM、GLM

- 跟GPT相似的Decoder架构，但是混合的注意力模式
- **相比Causal-LM只改了attention mask。**前缀部分是双向语言模型的标准架构，后面要生成的部分是单向的causal mask，即递归生成的单向语言模型

You take a pretrained model and train it on a domain dataset say, **legal contracts** or **medical notes**.

```

#### 2. Parameter-Efficient Fine-Tuning (PEFT / LoRA)

Fine-tuning large models is expensive. Enter **LoRA (Low-Rank Adaptation)** which is a way to train small adapters instead of updating the entire model. **LoRA（Low-Rank Adaptation，低秩适配）**是大模型的**参数高效微调方法**，冻结预训练主体权重，只训练少量低秩矩阵，用极低成本实现领域适配。

```
from peft import LoraConfig, get_peft_model
from transformers import AutoModelForCausalLM

model = AutoModelForCausalLM.from_pretrained("meta-llama/Llama-2-7b")
lora_config = LoraConfig(r=8, lora_alpha=32, target_modules=["q_proj","v_proj"])
model = get_peft_model(model, lora_config)
```

**LoRA fine-tuning** can be done on a **single GPU with 20% memory** use, making customisation affordable.

## 里程碑

大模型的发展历程中，有几个关键节点上的模型起到了里程碑式的作用，它们的技术创新深刻影响了整个领域的发展方向。

### 🧬 Transformer：奠定基石 (2017)

由Google在2017年提出，Transformer架构是整个大模型时代的开端。它的核心创新在于**自注意力机制（Self-Attention Mechanism）**。

- **告别序列依赖**：它完全摒弃了之前主流的循环神经网络（RNN）结构，解决了RNN无法并行计算和处理长距离依赖的痛点。
- **实现高效并行**：自注意力机制允许模型在处理一个词时，同时关注输入序列中的所有其他词，这使得训练过程可以高度并行化，效率得到数量级的提升。

可以说，没有Transformer，就没有今天的大模型繁荣景象。

### 💥 GPT-3：涌现能力的展示 (2020)

GPT-3将模型的参数规模推向了1750亿，并首次清晰地展示了“**涌现能力（Emergent Ability）**”。

- **少样本学习（Few-shot Learning）**：在此之前，模型通常需要针对特定任务进行大量数据微调。而GPT-3证明，当模型规模足够大时，仅需提供少量示例（甚至不提供），它就能理解任务意图并完成高质量的输出。这标志着模型从“专用工具”向“通用智能”迈出了关键一步。

- Sparse Attention，降低复杂度 O(n²) → O(n log n)

  Sparse Attention 稀释注意力,Sparse Attention 通过只让每个 token 关注局部邻居和部分远距离 token，将注意力计算复杂度从 O(n²) 降到 O(n·log n)，使 GPT-3 能够高效处理长序列文本,大幅增强了长文本处理能力,节省了计算资源用来处理更长的文本。

  > 稀疏注意力主要可以分为以下几类：
  >
  > #### **1. 基于固定模式 (Pattern-based)**
  >
  > 这是最直观的方法，预先设定好注意力关注的范围。
  >
  > - **局部窗口（Sliding Window）**：每个词只关注其前后固定距离内的邻居。这对于捕捉局部语法和短语非常有效。
  > - **全局标记（Global Tokens）**：指定少数几个特殊的“全局”词元（如句首的`[CLS]`标记），让所有词都能关注到它们，同时它们也能关注所有词，以此来保留一定的全局信息。
  >
  > #### **2. 动态稀疏 (Dynamic Sparse)**
  >
  > 这种方法更加智能，它不预设固定模式，而是根据每次具体的输入内容，实时计算出哪些部分是重要的。
  >
  > - **HISA (分层索引稀疏注意力)**：为了高效地从超长文本中找出关键信息，它采用了“先粗筛，再精选”的两步策略。首先将文本分块，快速筛选出最相关的几个文本块；然后只在被选中的小块内部进行精细的词元筛选。这种方法可以为DeepSeek等模型带来2-4倍的推理加速，且几乎不损失精度。
  > - **SALE (低比特估计稀疏注意力)**：为了在“检查精度”和“额外开销”之间取得平衡，它使用极低比特（如4-bit）量化来快速估算整个注意力图，然后用这个近似图来指导重要区域的选择。这种方法利用了GPU上低比特计算的高吞吐特性，能显著加速长文本的预填充阶段。
  >
  > #### **3. 可切换/混合注意力 (Switchable/Hybrid)**
  >
  > 这类方法旨在兼顾短文本的处理能力和长文本的效率。
  >
  > - **InfLLM-V2 (稠密-稀疏可切换注意力)**：它允许模型在遇到短序列时使用标准的稠密注意力，以保持最佳性能；而当序列长度超过阈值时，则无缝切换到高效的稀疏注意力模式。这种方式对模型结构改动极小，能够实现从短上下文到长上下文的平滑过渡，无需重新训练。

### ❤️ ChatGPT：引爆AI普及浪潮 (2022)

ChatGPT本身并非一个全新的基础模型，而是基于GPT-3.5系列，其里程碑意义在于成功应用并普及了一项关键技术：**基于人类反馈的强化学习（RLHF）**。

- **人机对齐（AI Alignment）**：通过收集人类对模型输出的偏好数据，训练一个奖励模型，再用这个奖励模型来指导语言模型的优化。这使得模型的输出更安全、更有用、更符合人类的指令和价值观，极大地提升了用户体验，让大模型真正走进了大众视野。

### 🧠 GPT-4：迈向多模态与架构革新 (2023)

GPT-4代表了技术上的又一次巨大飞跃，主要体现在两个方面：

1. **引入混合专家架构（MoE）**：这是一种稀疏化的模型架构。模型内部包含多个“专家”子网络，处理不同任务时，门控网络会根据输入内容，只激活最相关的几个专家进行处理。这种方式使得模型总参数量可以达到万亿级别，但每次推理的计算成本却可控，实现了性能与效率的兼顾。
2. **强大的多模态能力**：GPT-4展现了前所未有的图像理解和文本生成混合任务的能力，能够分析复杂的图表、解答视觉问题，推动了AI从纯文本处理向多感官理解的演进。

### ✨ Gemini：原生多模态的新范式 (2023起)

Gemini系列确立了一条与GPT不同的技术路线：**原生多模态（Native Multimodal）**。

- **统一预训练**：不同于GPT早期“先训练文本模型，再外挂视觉编码器”的方式，Gemini从预训练阶段开始，就同时在文本、图像、音频等多种模态的数据上进行学习。
- **无缝理解**：这种设计使其能从底层更自然、更统一地理解和推理跨模态信息，减少了信息损失，为处理更复杂的多媒体任务奠定了基础。

## Agent

当前 AI Agent（智能体）领域的发展已超越了早期的聊天机器人和简单的任务自动化，正朝着**持久化、自我进化、主动感知和多智能体协作**的方向快速演进。最新的方案不再仅仅关注模型本身的能力，而是致力于构建一个完整的、能融入实际工作流的生产级系统。

以下是几种代表性的前沿 Agent 方案及其核心创新点：

### 🚀 Hermes Agent：自我进化的开源先锋

Hermes Agent 是当前最受瞩目的开源项目之一，其核心理念是让 Agent “与你共同成长”。它通过一套精密的架构，实现了从被动执行到主动学习的转变。

- 主要创新点：
  1. **自动化学习闭环**：这是 Hermes 的灵魂。当它完成一个复杂任务后，会自动分析成功路径，将其提炼成一个结构化的“技能文件”（Skill File），并存储起来。下次遇到类似任务，它会直接调用这个技能，实现“越用越强”。
  2. **四层分级记忆系统**：为了支撑学习，它设计了模仿人类大脑的记忆架构，包括处理当前任务的短期工作记忆、关联任务信息的中期任务记忆、存储可复用技能的长期技能记忆，以及记录用户偏好的永久用户画像。这套系统基于 SQLite，能实现毫秒级检索，确保了跨会话的记忆连续性。
  3. **卓越的安全与生态**：在 OpenClaw 等早期框架因安全漏洞（如远程代码执行）备受困扰时，Hermes 凭借零 CVE 的安全设计赢得了开发者信任。同时，它对超过 200 种大模型的广泛支持，使其成为一个高度灵活的平台。

### 🔄 核心机制：从“执行”到“进化”的四步走

Hermes 的学习过程是一个自动化的流水线，不需要你手动去写代码教它，它会在后台悄悄完成以下步骤：

1. **执行与监控**
   当你给它一个复杂任务（比如写一个爬虫脚本）时，它会调用各种工具去尝试完成。在这个过程中，系统会全程监控它的操作步骤、工具调用次数以及最终结果。
2. **评估与反思**
   任务结束后，Hermes 不会直接“下班”。内置的评估器会给这次表现打分（1-10分）。
   - 如果任务失败了，它会分析错误日志，思考哪里出了问题。
   - 如果任务成功了，但步骤繁琐（比如调用了20次工具），它会思考：“下次能不能更简单点？”
3. **技能提炼**
   这是最关键的一步。一旦任务满足一定条件（例如评分≥7分，或者涉及复杂的步骤序列），Hermes 会自动触发“技能提取器”。
   它会将刚才成功的整个工作流——包括中间踩过的坑、用过的参数、修正过的代码——打包压缩成一个结构化的 Markdown 文件（即 **Skill File**）。这就好比它给自己写了一本“操作手册”或“错题本”。
4. **存储与复用**
   这个新生成的“技能文件”会被存入本地的技能库中。下次你再遇到类似任务时，它不再是从零开始推理，而是先检索技能库，直接调用这个现成的“操作手册”。
   - **效果：** 有用户实测，连续使用一个月后，同类任务的工具调用次数能从20多次压缩到8-10次，效率提升超过60%。

### 💾 记忆系统：像人一样分层管理知识

为了支撑这种学习，Hermes 没有采用简单的“全量存储”，而是设计了一套**四级分层记忆架构**，模仿人类大脑的处理方式：

| 记忆层级        | 功能描述                                                     | 类比                                                    |
| --------------- | ------------------------------------------------------------ | ------------------------------------------------------- |
| **L1 核心记忆** | 存储在 `MEMORY.md` 中，严格限制大小（约800 tokens），记录当前最重要的项目上下文。 | **短期工作区**：桌面上正在处理的文件。                  |
| **L2 用户画像** | 存储在 `USER.md` 中，记录你的偏好、沟通风格和技术栈习惯。    | **个人习惯**：记得你不喜欢废话，或者偏爱 Python。       |
| **L3 长时记忆** | 使用 SQLite + FTS5 全文检索，存储所有历史对话。只有被搜索到时才会加载。 | **图书馆档案**：平时不占脑子，需要时能瞬间查到。        |
| **L4 技能记忆** | 存储自动生成的技能文件。只有在匹配到相关任务时才加载具体内容。 | **肌肉记忆/工具箱**：学会骑车后就不用再想怎么蹬踏板了。 |



### ☁️ Cloudflare Project Think：将 Agent 变为基础设施

Cloudflare 提出的 Project Think 代表了另一种思路：将 Agent 从运行在本地的脆弱进程，升级为跑在互联网上的、持久且可靠的“基础设施”。

- 主要创新点：
  1. **Agent 作为 Serverless 服务**：它利用 Durable Objects 等技术，让 Agent 可以持久化运行数周，即使闲置成本也为零，并且能从网络中断或故障中自动恢复。这解决了传统 Agent“关机就断”的痛点。
  2. **跨休眠的持久记忆**：Think 引入了“上下文块”（Context Blocks）的概念，允许模型主动读取和更新结构化的记忆片段。这些记忆与 Agent 本身一起被持久化，不依赖任何外部数据库，使得 Agent 拥有了真正的长期记忆。
  3. **极致的效率优化**：通过非破坏性压缩技术，旧的对话消息会被总结而非删除，既节省了上下文窗口，又保留了完整的历史记录可供随时检索，大幅降低了 Token 消耗和成本。

### 🏢 ThinkingAI Agentic Engine：企业级的“感-知-行”闭环

ThinkingAI 推出的 Agentic Engine 专注于解决企业落地难题，它将 Agent 的理念从“个人助手”转变为能为企业创造价值的“团队协作中枢”。

- 主要创新点：
  1. **“感-知-行”三位一体**：该平台强调 Agent 应具备**全域感知**（主动发现业务异常）、**深度理解**（结合企业内部知识分析原因）和**行动闭环**（自主执行策略，如调整预算、发起A/B测试）的能力，形成一个无需人工干预的智能闭环。
  2. **多智能体协同作战 (CoWork)**：它不是提供单个超级助手，而是组建一支由数据分析 Agent、A/B实验 Agent、智能运营 Agent 等构成的专业团队。这些 Agent 各司其职，通过统一的编排器进行调度和协作，共同完成复杂的业务流程。
  3. **深度融合业务知识**：Agentic Engine 的核心壁垒在于其内置的行业知识库（Know-how）。它将企业的隐性知识（如“留存”的计算口径）结构化，使通用大模型能够快速转变为懂业务的专家，解决了“最后一公里”的应用难题。

### 🧬 Agent-R1：强化学习驱动的主动探索

由中国科学技术大学团队提出的 Agent-R1 框架，则从训练方法上进行了革新，旨在赋予 Agent 更强的主动性和规划能力。

- 主要创新点：
  1. **过程奖励机制**：传统的强化学习通常只在任务最终完成时才给予奖励。Agent-R1 创新地引入了“过程奖励”，在 Agent 完成有效搜索、信息整合等中间步骤时就给予即时反馈。这极大地提升了学习效率，让 Agent 能更快地掌握复杂任务的拆解和执行。
  2. **主动探索能力**：该框架鼓励 Agent 像侦探一样，主动调用工具、探索环境以获取信息，而不是被动地等待用户指令。这种设计使其在面对模糊问题时，表现出更强的独立思考和解决问题的能力。
